# Agent Word Processor 打包指南

本程序是一个基于AI的Word文档批量处理工具，需要Python运行环境。以下提供三种打包方案，解决在没有Python环境的机器上运行的问题。

## 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| PyInstaller | 单文件，易分发 | 文件较大(~50MB) | 桌面应用分发 |
| 便携式Python | 完整环境，灵活 | 体积大(~100MB) | 开发/测试环境 |
| Docker容器 | 标准化，隔离 | 需要Docker环境 | 服务器部署 |

## 方案一：PyInstaller 打包（推荐）

### 特点
- 将程序打包成单个可执行文件
- 包含Python解释器和所有依赖
- 适合最终用户分发

### 使用步骤

1. **安装依赖**
```bash
pip install -r requirements.txt
```

2. **执行打包**
```bash
python build_spec.py
```

3. **输出文件**
- 可执行文件：`dist/AgentWordProcessor`
- Windows：`AgentWordProcessor.exe`
- Linux/Mac：`AgentWordProcessor`

4. **分发说明**
- 将 `dist` 文件夹整个复制给用户
- 用户双击可执行文件即可运行
- 首次运行会创建配置文件，需要编辑API密钥

### 注意事项
- Windows可能会出现杀毒软件误报，添加信任即可
- 首次启动可能较慢（解压临时文件）

## 方案二：便携式Python环境

### 特点
- 包含完整的Python运行环境
- 用户可以看到所有源代码
- 支持用户自定义修改

### 使用步骤

1. **执行打包脚本**
```bash
python build_portable.py
```

2. **输出目录结构**
```
portable_build/
├── python/              # Python运行环境
├── agent core.py        # 主程序
├── agent_config.json    # 配置文件
├── 启动程序.bat         # Windows启动脚本
├── start.sh            # Linux/Mac启动脚本
└── 使用说明.txt         # 用户说明
```

3. **分发方式**
- 将整个 `portable_build` 文件夹压缩
- 用户解压后运行启动脚本

### 优点
- 环境完整，兼容性好
- 用户可以自由修改代码
- 支持添加新的Python模块

## 方案三：Docker容器化

### 特点
- 标准化的运行环境
- 完全隔离，避免环境冲突
- 适合服务器部署

### 使用步骤

1. **构建镜像**
```bash
docker build -t agent-word-processor .
```

2. **运行容器**
```bash
# 使用docker run
docker run -it --rm \
  -v $(pwd)/agent_config.json:/app/config/agent_config.json:ro \
  -v $(pwd)/data:/app/data \
  agent-word-processor

# 或使用docker-compose
docker-compose up
```

3. **配置说明**
- 配置文件需要挂载到容器内
- 数据目录映射到宿主机保存结果

## 推荐方案选择

### 桌面用户分发
**推荐：PyInstaller方案**
- 用户体验最好，双击即用
- 无需额外安装依赖
- 适合非技术用户

### 开发测试环境
**推荐：便携式Python方案**
- 环境完整，便于调试
- 可以方便修改代码
- 支持添加新功能

### 服务器部署
**推荐：Docker方案**
- 环境标准化
- 便于维护和更新
- 资源隔离

## 常见问题解决

### Python路径问题
程序中的Python代码执行使用了 `sys.executable`，确保使用正确的Python解释器路径。

### 依赖包问题
如果遇到导入错误，检查以下依赖是否完整：
- openai
- lxml  
- Pillow

### 文件权限问题
Linux/Mac系统需要确保启动脚本有执行权限：
```bash
chmod +x start.sh
```

### API配置问题
确保 `agent_config.json` 中的API密钥配置正确：
```json
{
  "api_pool": ["your-actual-api-key"],
  "current_position": 0,
  "base_url": "https://api.siliconflow.cn/v1",
  "model": "Qwen/Qwen3-235B-A22B-Instruct-2507"
}
```

## 性能优化建议

1. **减小包体积**
   - 移除不必要的依赖
   - 使用 `--exclude-module` 排除不需要的模块

2. **加快启动速度**
   - 使用 `--onedir` 代替 `--onefile`
   - 预热常用模块

3. **内存优化**
   - 适当调整 `max_tokens` 参数
   - 控制并发请求数量

通过以上方案，您可以将程序打包成独立运行的版本，解决Python环境依赖问题。
